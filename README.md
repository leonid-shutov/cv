# Aggregator

**Stack**: React, Redux, GraphQL, MongoDB, Gatsby, Next.js, Apollo, Redis, Google Cloud,
CircleCi, Contentful, DataDog

**Description**: React application that serves more than 1000 landing pages from the CMS and a long form composed of questions coming from a big variety of partners. The main challenge of the form was to create a no-code solution that would allow configuring questions in a JSON format keeping React components shared between different countries and integrations. To achieve this, we used metaprogramming techniques to define the behavior of the form dynamically, streamlining the configuration process and enhancing flexibility

# Web Scraper

**Stack**: Node.js, Puppeteer, Redis, BullMQ, MongoDB

**Description**: The goal of the project was to automate the process of submitting long dynamic forms consisting of several pages and getting quotes. One of the main challenges of the project was to ensure that the scraper could serve multiple connections at the same time without overwhelming the server. To address this issue, I implemented a queue with multiprocessing, which used Redis to manage the queue of requests. The tech stack also included MongoDB to store the scraped data with screenshots, and Puppeteer to interact with the browser in headless mode
